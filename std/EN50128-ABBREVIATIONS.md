# EN 50128:2011 Official Abbreviations

**Source:** EN 50128:2011 Section 3.2  
**Location:** `std/EN50128-2011.md` lines 991-1041

This document contains the official abbreviations defined in EN 50128:2011. All project documentation SHALL use these abbreviations consistently.

## Usage Guidelines

1. **Consistency:** All project documents SHALL use these official abbreviations
2. **First Use:** Define abbreviations on first use in each document: "Software Requirements Specification (SRS)"
3. **Section 3.2:** When in doubt, refer to EN 50128:2011 Section 3.2
4. **Role Names:** Use full role names in prose, abbreviations in tables and diagrams
5. **Commands:** OpenCode agent commands may use shorter abbreviations (e.g., `/req`, `/des`, `/imp`) for usability
